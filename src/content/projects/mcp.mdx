# Model Context Protocol (MCP) Server for Dev.to Publishing

A production-grade Python tool implementing Anthropic's **Model Context Protocol (MCP)**. Exposes specialized blog publishing integrations as native, schema-validated tools directly invocable by IDE AI agents (e.g., Claude Desktop, Cursor).

---

**Key Protocol Paradigm:** Rather than making manual API calls, the client-facing LLM reads the tool schemas published by the MCP server, binds arguments to a typed call payload, and executes the python tool natively over standard IO.

---

## Engineering Design Review & Decisions

### 1. Unified stdio Protocol Interface
* **Decision:** We implemented standard input/output (`stdio`) as the primary transport protocol channel rather than HTTP/WebSockets.
* **Rationale:** Establishing a local CLI runtime transport loop avoids port allocation conflicts on local development machines, completely bypasses external firewall constraints, and ensures low-latency execution directly managed by the host client application (e.g. Claude Desktop).

### 2. High-Performance Packaging with `uv`
* **Decision:** Replaced traditional pip virtual environments with Astral's `uv` tool suite for dependency locked synchronization.
* **Rationale:** `uv` leverages a Rust-based toolchain that executes package synchronizations under **150ms** and compiles isolated execution environments in sub-second timelines. This guarantees rapid, reproducible local environment setup for developers deploying the MCP server.

### 3. API Key Sandboxing
* **Decision:** Bounded critical publishing authority entirely within standard local environment variables (`DEVTO_API_KEY`) loaded dynamically.
* **Rationale:** Ensures no operational tokens are exposed during tool schemas initialization and allows developers to swap publishing access profiles dynamically without modifying the underlying tool registration script.

## Tool Capabilities & Specifications

### Exposed Native Tools
* `publish_blog_to_devto` — Programmatically drafts and publishes markdown assets directly to Dev.to.
  * **Input Payload Constraints:** Requires `title`, `body_markdown`, `tags` (array of strings), and an optional `series` identifier.
